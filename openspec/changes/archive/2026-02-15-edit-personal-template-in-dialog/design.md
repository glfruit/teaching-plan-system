## Context

系统已具备个人模板的创建、查询、删除、覆盖套用能力，且后端提供模板更新接口。缺口主要在前端交互：教师无法加载并编辑已保存模板内容。

## Goals / Non-Goals

**Goals:**
- 在模板面板内提供弹窗编辑能力。
- 编辑时从模板原始内容加载，允许修改标题与整份模板正文。
- 保存时复用现有模板更新 API，并保持 JSON-first 内容一致性。
- 取消时保证模板不变。

**Non-Goals:**
- 不新增独立模板编辑页面。
- 不新增模板版本历史、审计、协作编辑。

## Decisions

### 1) 采用同页弹窗编辑
- 原因：改动最小，不打断当前教案编写流程。
- 备选：独立页面（更重，切换成本高）。

### 2) 编辑内容复用现有教案表单结构
- 原因：可直接复用 `mapFetchedPlanToForm` 与 `buildPlanPayload`，减少字段映射和遗漏风险。

### 3) 保存走现有 `updateTemplate`（PATCH）
- 原因：后端能力已具备，避免引入新接口与额外测试面。

## Risks / Trade-offs

- [误编辑风险] -> 弹窗“取消”显式关闭且不提交；仅“保存修改”触发更新。
- [状态不同步风险] -> 保存成功后刷新模板列表，避免本地陈旧数据。
